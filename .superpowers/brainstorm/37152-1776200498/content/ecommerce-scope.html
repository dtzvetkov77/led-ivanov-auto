Каква е функционалността на магазина?

Това определя голяма част от архитектурата — кликни на предпочитания вариант

Каталог + Контакт
✓ Преглед на продукти и цени
✓ Филтриране по марка/модел
✓ WhatsApp / Viber / Форма за поръчка
✗ Няма количка / плащане онлайн
✗ Няма checkout процес
Поръчай по Viber / Телефон

A — Каталог с контакт за поръчка

Показваш продукти и цени, но поръчките стават по телефон/Viber (както сега). По-просто, по-бързо за изграждане. Подходящо ако повечето клиенти се свързват директно.

Пълен Онлайн Магазин
✓ Количка и checkout
✓ Плащане с карта (Stripe) + наложен платеж
✓ Регистрация / история на поръчки
✓ Имейл потвърждения
✓ Админ панел за управление

B — Пълен e-commerce

Пълноценен онлайн магазин с количка, плащане с карта, наложен платеж и история на поръчки. По-сложно, но дава пълна автономия и по-добро UX за клиентите.

Хибрид
✓ Каталог с всички продукти и цени
✓ "Добави в количка" → форма за поръчка
✓ Наложен платеж (Еконт / Спиди)
✓ Email + Viber нотификации
~ Онлайн плащане — по-късно

C — Количка + наложен платеж (без карта)

Средният вариант — клиентът добавя в количка и поръчва, но плаща при доставка (Еконт/Спиди). Без интеграция с платежна система. Лесно за разширяване по-късно.
